// testing process

How I Approach Quality

01
Requirements Review
Carefully read user stories, acceptance criteria, and feature specs to understand what "done" looks like before touching the app.
02
Test Planning
Define scope, strategy, entry/exit criteria, risk areas, and test data requirements. Produce a test plan document for alignment.
03
Test Case Design
Write positive, negative, and edge case scenarios for each feature. Include data variations, boundary conditions, and error paths.
04
Test Execution
Systematic execution across all required browsers and devices. Mark pass/fail, capture evidence for any failures.
05
Bug Reporting
Clear, developer-friendly bug reports with steps to reproduce, expected vs actual results, severity, and annotated screenshots.
06
Regression & Sign-off
After fixes, verify bugs are resolved and check for side effects. Produce a final report confirming release readiness.
